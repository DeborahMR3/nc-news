GET /api

Returns a JSON object describing all available endpoints.

Example response

{
  "endpoints": {
    "/api": "...",
    "/api/topics": "...",
    
  }
}

GET /api/topics

Returns an array of topic objects. Each object contains:

  • slug – the topic name
  • description – a short description

GET /api/articles

Returns an array of article objects with comment_count.

Query parameters

  • topic – filter by topic
  • sort_by – column to sort (default created_at)
  • order – asc or desc (default desc)
  • limit – results per page (default 10)
  • p – page number (default 1)

GET /api/articles/:article_id

Returns a single article object matching :article_id.

PATCH /api/articles/:article_id

Increments an article's vote count.

Request body

{ "inc_votes": 1 }

The endpoint responds with the updated article object.

GET /api/articles/:article_id/comments

Returns an array of comments for the specified article.

Query parameters

  • limit – results per page (default 10)
  • p – page number (default 1)

POST /api/articles/:article_id/comments

Creates a new comment on the specified article.

Request body

{
  "username": "butter_bridge",
  "body": "Great article!"
}

The endpoint responds with the posted comment.

PATCH /api/comments/:comment_id

Updates a comment's vote count.

Request body

{ "inc_votes": -1 }

Responds with the updated comment.

DELETE /api/comments/:comment_id

Deletes the specified comment. Responds with status code 204 No Content.

GET /api/users

Returns an array of user objects.

GET /api/users/:username

Returns a single user object matching :username.
